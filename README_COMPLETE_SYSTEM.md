## 🚀 How to Use This System

### Week 1: Preparation
```
1. Read EPISODE_OUTLINE_SOUL_OR_SEVERANCE.md completely
2. Gather cast (Hal, Ada, VERA voice)
3. Prepare sound effects and music
4. Review FINAL_CHECKLIST.md pre-production section
5. Brief team on roles and expectations
```

### Week 2: Recording
```
1. Follow FINAL_CHECKLIST.md recording session section
2. Record 42-minute episode per the outline
3. Edit and mix (target: 35–42 minutes)
4. QA pass (timing, clarity, effects)
5. Prepare git commit of SOUL.md files
```

### Week 3: Publishing
```
1. Export final audio
2. Write episode description (use LISTENER_GUIDE as source)
3. Commit SOUL.md files to git (see DELIVERY_SUMMARY for commit message)
4. Upload episode to hosting platform
5. Schedule social media posts (see LISTENER_GUIDE for teasers)
6. Send email to subscribers
```

### After Publication: Monthly Cadence
```
1. Let 4 episodes accumulate (don't rush)
2. Luis reviews post-episode analysis (15 min)
3. Luis makes ~1 decision (deprecate/promote/monitor/growth)
4. If decision: update SOUL.md, commit, record patch notes
5. If no decision: do nothing (default is "no patch")
6. VERA Patch Notes air only when real changes happened
```
